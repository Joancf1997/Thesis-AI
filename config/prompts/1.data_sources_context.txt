### USER SEGMENTATION DATA SOURCE CONTEXT

The system includes a data source called **User Segmentation Dataset**, which contains groups of readers clustered according to their content consumption patterns, engagement behavior, regional activity, and topical preferences.

Each **user segment** represents a unique behavioral profile derived from the news platform’s analytics data.

This data source provides the foundation for the user-segment-related analytical tools (e.g., get_segment_description, get_segment_time_activity, get_segment_regions, etc.).

Each segment includes the following key attributes:
- **id (int)**: Unique numeric identifier for the segment.
- **title (string)**: Descriptive name summarizing the main interests of the segment.
- **description (string)**: Narrative text describing the segment’s reading behavior and preferences.
- **docs_engaged / docs_notengaged**: Lists of articles that users engaged or did not engage with.
- **high_docs / med_docs / low_docs**: Documents ranked by representativeness of the segment.
- **df**: DataFrame with event-level data (time, engagement, topics).
- **regions**: Dictionary mapping geographic regions to user counts.
- **user_type_cnt**: Dictionary with counts of “frequent” and “nonfrequent” users.
- **seq_model**: Probabilistic model describing topic transitions and next-topic predictions.
- **regions_desc**: Summary text of regional activity.

### NEWS TOPICS DATA SOURCE CONTEXT

The system includes a data source called **News Topics Dataset**, which contains the latent topics discovered from the entire corpus of news articles through topic modeling.  
Each **topic** represents a group of articles that share similar themes or semantic content.  
These topics are the foundation for interpreting user interests, engagement patterns, and cross-topic transitions in the system.

This data source provides input to topic-related analytical tools (e.g., get_news_topics_info, get_news_topics_high_docs, get_news_topics_low_docs).

Each topic entry includes the following key attributes:
- **id (int)**: Unique identifier of the topic.
- **title (string)**: Concise and descriptive name of the topic, summarizing its main focus.
- **desc (string)**: Narrative description of what the topic represents, including the key themes or issues covered.
- **high_docs (list)**: List of article IDs that are highly representative of this topic.
- **med_docs (list)**: Articles with moderate association to the topic.
- **low_docs (list)**: Articles that are weakly associated with the topic but still relevant for broader analysis.

### RAW NEWS ARTICLES DATA SOURCE CONTEXT

The system includes a data source called **Raw News Articles Dataset**, which contains detailed metadata and text information for all published news articles.  
It is the most granular source of information in the system and serves as the foundation for topic modeling, segment analysis, and content retrieval.  
Although it provides article-level data, most analytical operations use it indirectly through higher-level abstractions such as user segments or topic clusters.

This dataset is primarily used by tools that need to access article titles, teasers, publication dates, or cluster associations (e.g., get_articles_info, get_top_recent_articles, get_unique_clusters).

---

### DATA STRUCTURE OVERVIEW

Each row represents one article with rich metadata, textual fields, and derived analytical columns.  
The dataset includes approximately **68,000 records** and the following columns:

| Column | Description |
|:--|:--|
| **primaryCategory** | The main editorial category of the article (e.g., *bayern*, *sport*, *wirtschaft*). |
| **first_publication_date** | Unix timestamp (in milliseconds) representing the first time the article was published. |
| **title** | The article headline. |
| **teaserText** | Short teaser or preview text shown before reading the article. |
| **tags** | List of keywords and metadata tags assigned to the article. |
| **text** | The raw article text in HTML or structured format. |
| **text_clean** | Preprocessed version of the text for analytical and modeling tasks (cleaned and tokenized). |
| **embedding** | Numerical vector representation (embedding) of the article text for similarity or clustering tasks. |
| **word_count** | Total number of words in the article. |
| **read_time_seconds** | Estimated reading time based on text length and complexity. |
| **tags_x / merged_doc** | Processed metadata combining tags and document text for unified analysis. |
| **region** | List of regional labels associated with the article (e.g., *Oberbayern*, *Schwaben*). |
| **clusters** | List of cluster or topic IDs assigned to this article (corresponding to topics in the news_topics dataset). |
| **_d columns (title_d, text_d, teaser_d, etc.)** | Duplicated or preprocessed versions of the main text fields for modeling and debugging. |

### ROLE IN THE ANALYTICAL SYSTEM

This dataset underpins all higher-level analyses by providing:
- Article metadata for **topic labeling** and **temporal analysis**.
- Source data for **user–article interactions** in the user segmentation dataset.
- A bridge between **topics**, **segments**, and **geographical context**.
- Factual information retrieval when the agent must **display article titles, teasers, or publication dates**.
